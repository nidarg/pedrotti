# `app/globals.css`

```css
@import "tailwindcss";

:root {
  --background: #ffffff;
  --foreground: #171717;
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}
```

## Ce face fișierul

Acesta este fișierul global de stiluri. El se aplică peste toată aplicația.

## Explicație pe bucăți

### `@import "tailwindcss";`

Încarcă Tailwind în aplicație. Fără asta, clasele Tailwind nu ar funcționa.

### `:root { ... }`

Aici definești variabile CSS globale:

- `--background`
- `--foreground`

Aceste variabile sunt folosite apoi pentru culoarea de fundal și culoarea textului.

### `@theme inline { ... }`

Leagă variabilele tale globale de sistemul de theme folosit în proiect.

Practic:

- `--color-background` primește valoarea din `--background`
- `--color-foreground` primește valoarea din `--foreground`
- `--font-sans` și `--font-mono` primesc fonturile încărcate în `layout.tsx`

### `@media (prefers-color-scheme: dark)`

Dacă userul are dark mode activat în sistem, schimbi automat valorile:

- fundalul devine închis
- textul devine deschis

### `body { ... }`

Aici spui că întregul document folosește:

- fundalul din variabila `--background`
- culoarea textului din variabila `--foreground`
- font fallback `Arial, Helvetica, sans-serif`

# `app/layout.tsx`

```tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";
import { GoogleAnalytics } from "@next/third-parties/google";
import { GA_MEASUREMENT_ID } from "@/lib/gtag";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html
      lang="en"
      className={`${geistSans.variable} ${geistMono.variable} h-full antialiased`}
    >
      <body className="min-h-full flex flex-col">{children}</body>
      <GoogleAnalytics gaId={GA_MEASUREMENT_ID} />
    </html>
  );
}
```

## Ce este acest fișier

Acesta este layout-ul global al aplicației Next.js App Router.

Tot ce se randază în aplicație este învelit de acest layout.

## Explicație pe cod

### `import type { Metadata } from "next";`

Importă tipul TypeScript pentru metadata paginii.

### `import { Geist, Geist_Mono } from "next/font/google";`

Încarcă fonturi Google direct prin Next.js, optimizat.

### `import "./globals.css";`

Aplică stilurile globale în toată aplicația.

### `import { GoogleAnalytics } from "@next/third-parties/google";`

Importă componenta oficială Next.js pentru integrarea Google Analytics.

### `import { GA_MEASUREMENT_ID } from "@/lib/gtag";`

Ia ID-ul GA4 din fișierul tău centralizat.

---

## Fonturile

```tsx
const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});
```

Aici creezi fontul principal și îi spui să fie expus ca variabilă CSS: `--font-geist-sans`.

La fel pentru mono:

```tsx
const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});
```

## `metadata`

```tsx
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};
```

Asta setează metadata implicită pentru pagină:

- titlul
- descrierea

În varianta finală, aici ar trebui schimbate cu date Pedrotti.

## `RootLayout`

```tsx
export default function RootLayout({ children }) { ... }
```

Este componenta principală de layout.

### Parametrul `children`

`children` reprezintă tot conținutul paginii curente. În cazul tău, `page.tsx` va fi injectat aici.

---

## HTML-ul returnat

### `<html lang="en" ...>`

Setează limba documentului. La tine este `en`, dar pentru site-ul Pedrotti ar trebui probabil `it`.

### `className={`${geistSans.variable} ${geistMono.variable} ...`}`

Atașează variabilele CSS pentru fonturi pe elementul `html`, astfel încât ele să poată fi folosite în stiluri.

### `<body>{children}</body>`

Aici intră efectiv conținutul paginii.

### `<GoogleAnalytics gaId={GA_MEASUREMENT_ID} />`

Injectează scriptul Google Analytics în aplicație, folosind ID-ul din `gtag.ts`.

## De ce are `"use client"`

Pentru că folosește:

- `onClick`
- tracking analytics
- componentă interactivă cu geolocation

În App Router, fără `"use client"`, componenta ar fi server component și nu ar putea folosi interacțiuni din browser.

---

## Explicație pe cod

### `const whatsappMessage = encodeURIComponent(...)`

Construiești mesajul implicit pentru WhatsApp.

Folosești `encodeURIComponent` ca textul să fie sigur pentru URL. Spațiile și caracterele speciale sunt transformate în format valid pentru query string.

### `const phoneLink = \`tel:${siteData.company.phoneHref}\`;`

Construiești linkul de apel telefonic.

Exemplu rezultat:

```txt
tel:+390464913290
```

Când userul apasă, telefonul încearcă să inițieze apelul.

### `const whatsappLink = ...`

Construiești linkul către WhatsApp Web / app.

Formatul folosit este:

```txt
https://wa.me/NUMAR?text=MESAJ
```

---

## Primul buton: Call

```tsx
<a href={phoneLink} onClick={() => trackEvent("phone_click", { placement: "hero" })}>
```

### Ce face

- deschide apelul telefonic
- trimite event în GA4 înainte/în momentul clickului

### `trackEvent("phone_click", { placement: "hero" })`

Aici marchezi că s-a făcut click pe apel și specifici unde s-a întâmplat: în hero.

Asta este util pentru comparație cu alte locuri din site.

---

## Al doilea buton: WhatsApp

```tsx
<a href={whatsappLink} target="_blank" rel="noreferrer" ...>
```

### Ce face

- deschide WhatsApp într-un tab nou sau în aplicație
- precompletează mesajul
- trimite analytics event

### `target="_blank"`

Deschide într-un context nou.

### `rel="noreferrer"`

E o măsură de siguranță și privacy pentru linkurile externe deschise în tab nou.

---

## Al treilea buton: `LocationWhatsAppButton`

Aici nu mai scrii logica direct în `HeroActions`, ci o muți într-o componentă separată.

Asta este bine deoarece:

- logica este reutilizabilă
- nu duplici cod
- păstrezi componenta curată


## `Image fill`

```tsx

  src={image.src}
  alt={image.alt}
  fill
  sizes="(max-width: 639px) 100vw, 50vw"

```

### `fill`

Spune imaginii să ocupe complet containerul părinte.

Ca asta să funcționeze, containerul părinte trebuie să aibă `position: relative`, ceea ce ai făcut.

### `sizes`

Îi spune browserului cât spațiu va ocupa imaginea în funcție de viewport. Este important pentru performanță.

```tsx
};

export default function LocationWhatsAppButton({
  className = "",
  iconSize = 20,
  label = "Invia posizione",
}: LocationWhatsAppButtonProps) {
  const [isLoading, setIsLoading] = useState(false);

  const handleSendLocation = () => {
    trackEvent("location_click", {
      placement: label,
    });

    if (!navigator.geolocation) {
      const fallbackMessage = encodeURIComponent(
        "Ciao, ho bisogno di soccorso stradale. Non riesco a condividere automaticamente la mia posizione. Vi invio la posizione manualmente su WhatsApp."
      );

      window.open(
        `https://wa.me/${siteData.company.whatsappNumber}?text=${fallbackMessage}`,
        "_blank",
        "noreferrer"
      );
      return;
    }

    setIsLoading(true);

    navigator.geolocation.getCurrentPosition(
      (position) => {
        const latitude = position.coords.latitude;
        const longitude = position.coords.longitude;

        const mapsLink = `https://www.google.com/maps?q=${latitude},${longitude}`;

        const message = encodeURIComponent(
          `Ciao, ho bisogno di soccorso stradale. Questa è la mia posizione: ${mapsLink}. Se necessario posso inviare anche foto del veicolo e targa.`
        );

        window.open(
          `https://wa.me/${siteData.company.whatsappNumber}?text=${message}`,
          "_blank",
          "noreferrer"
        );

        setIsLoading(false);
      },
      () => {
        const fallbackMessage = encodeURIComponent(
          "Ciao, ho bisogno di soccorso stradale. Non sono riuscito a condividere automaticamente la mia posizione. Vi invio la posizione manualmente su WhatsApp."
        );

        window.open(
          `https://wa.me/${siteData.company.whatsappNumber}?text=${fallbackMessage}`,
          "_blank",
          "noreferrer"
        );

        setIsLoading(false);
      },
      {
        enableHighAccuracy: true,
        timeout: 10000,
        maximumAge: 0,
      }
    );
  };

  return (
    <button
      type="button"
      onClick={handleSendLocation}
      disabled={isLoading}
      className={className}
    >
      <MapPinned size={iconSize} />
      {isLoading ? "Recupero posizione..." : label}
    </button>
  );
}
```

# `lib/gtag.ts`

```ts
export const GA_MEASUREMENT_ID = "G-11N01GEZSR";

declare global {
  interface Window {
    gtag: (
      command: "event",
      eventName: string,
      params?: Record<string, string | number | boolean>
    ) => void;
  }
}

export const trackEvent = (
  eventName: string,
  params?: Record<string, string | number | boolean>
) => {
  if (typeof window === "undefined" || !window.gtag) return;

  window.gtag("event", eventName, params);
};
```

## Ce face

Centralizează integrarea cu Google Analytics 4.

---

## `GA_MEASUREMENT_ID`

Aici păstrezi ID-ul de tracking.

Este folosit în `layout.tsx`.

---

## `declare global { interface Window { gtag: ... } }`

Spui TypeScript-ului că obiectul global `window` are o funcție `gtag`.

Fără asta, TypeScript ar da eroare pentru `window.gtag`.

---

## `trackEvent(...)`

Este un helper reutilizabil.

În loc să scrii peste tot:

```ts
window.gtag("event", "phone_click", {...})
```

scrii simplu:

```ts
trackEvent("phone_click", {...})
```

---

## Verificarea de siguranță

```ts
if (typeof window === "undefined" || !window.gtag) return;
```

### De ce e importantă

- dacă rulează pe server, `window` nu există
- dacă GA nu s-a încărcat încă, `window.gtag` poate lipsi

În ambele cazuri eviți crash-ul aplicației.

Codul tău este structurat bine pentru un MVP:

- ai separare clară între layout, componente și date
- ai logică reutilizabilă
- ai tracking centralizat
- ai o componentă utilă și bine gândită pentru geolocation + WhatsApp

Din punct de vedere arhitectural, cele mai importante piese sunt:

- `page.tsx` ca orchestrator
- `site-data.ts` ca sursă centrală
- `LocationWhatsAppButton.tsx` ca logică principală de conversie
- `gtag.ts` ca helper de analytics
